    **1. Create `components/engagements/merge-suggestion-section.tsx`:**

    `'use client'` component. Props: `{ fork: ForkedPromptWithTitle, engagementId: string }`.

    State management:
    - `const [mergeStatus, setMergeStatus] = useState(fork.merge_status ?? 'none')`
    - `const [mergeNote, setMergeNote] = useState(fork.merge_suggestion ?? '')`
    - `const [declineReason] = useState(fork.merge_decline_reason ?? '')`
    - `const [dialogOpen, setDialogOpen] = useState(false)`
    - `const [isPending, startTransition] = useTransition()`

    Status badge config object:
    ```typescript
    const statusConfig = {
      pending: { bg: 'bg-[#FFB852]/15', text: 'text-[#FFB852]', icon: Clock, label: 'Pending Review' },
      approved: { bg: 'bg-[#65CFB2]/15', text: 'text-[#65CFB2]', icon: Check, label: 'Merged' },
      declined: { bg: 'bg-[#E3392A]/15', text: 'text-[#E3392A]', icon: X, label: 'Declined' },
    }
    ```

    `handleSubmit` function:
    - Calls `startTransition(async () => { ... })`
    - Invokes `suggestMerge(fork.id, mergeNote)`
    - On success: `setMergeStatus('pending')`, `setDialogOpen(false)`, `toast.success('Merge suggestion submitted')`
    - On error: `toast.error("Couldn't save changes. Try again.")`

    Render logic driven by `mergeStatus`:

    **When `mergeStatus === 'none'` or `mergeStatus === null`:**
    - Section heading: `<span className="text-[13px] text-muted-foreground">Merge suggestion</span>`
    - Button: `<Button variant="outline" className="w-full" onClick={() => setDialogOpen(true)}>` with `<GitMerge className="size-4 mr-2" />` + "Suggest Merge"
    - Dialog (using shadcn `Dialog`, NOT AlertDialog):
      - `<DialogTitle>Suggest a merge</DialogTitle>`
      - `<DialogDescription>Your adaptation notes, rating, and feedback will be shared with the reviewer automatically.</DialogDescription>`
      - `<Textarea placeholder="Why should this be merged back?" value={mergeNote} onChange={...} className="min-h-[80px] resize-none" />`
      - Footer: ghost button "Discard suggestion" (closes dialog) + primary button "Submit suggestion" (calls handleSubmit, disabled when `mergeNote.trim() === ''` or `isPending`, shows `Spinner` when isPending)

    **When `mergeStatus === 'pending'`:**
    - Section heading same as above
    - Status badge: `<span className="inline-flex items-center gap-1 px-2 py-0.5 rounded-md text-[13px] bg-[#FFB852]/15 text-[#FFB852]"><Clock className="size-3" />Pending Review</span>`

    **When `mergeStatus === 'approved'`:**
    - Same heading
    - Status badge with teal: `bg-[#65CFB2]/15 text-[#65CFB2]`, `<Check className="size-3" />`, "Merged"

    **When `mergeStatus === 'declined'`:**
    - Same heading
    - Status badge with red: `bg-[#E3392A]/15 text-[#E3392A]`, `<X className="size-3" />`, "Declined"
    - Below badge: decline reason text `<p className="text-[13px] text-muted-foreground italic mt-2">{declineReason}</p>`
    - Below reason: `<button className="text-[13px] text-[#4287FF] hover:underline mt-1" onClick={() => { setMergeStatus('none'); setDialogOpen(true) }}>Revise & resubmit</button>`

    Imports: `useState`, `useTransition` from react; `toast` from sonner; `GitMerge`, `Clock`, `Check`, `X` from lucide-react; `Button` from `@/components/ui/button`; `Dialog`, `DialogContent`, `DialogHeader`, `DialogTitle`, `DialogDescription`, `DialogFooter` from `@/components/ui/dialog`; `Textarea` from `@/components/ui/textarea`; `Spinner` from `@/components/ui/spinner`; `suggestMerge` from fork actions; `ForkedPromptWithTitle` type from fork types.
