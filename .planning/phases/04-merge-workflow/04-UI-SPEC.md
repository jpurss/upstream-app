## Interaction Contracts

### Merge Suggestion Section (Fork Sidebar ŌĆö Section 8)

**Trigger state (no suggestion yet):**
- Label: "Merge suggestion" (13px, muted, sidebar section heading pattern)
- Button: "Suggest Merge" ŌĆö `variant="outline"` full width, 14px, icon: `GitMerge` from lucide
- Positioned after Section 7 (Forked date), preceded by `border-t border-border`

**Dialog (on button click):**
- `Dialog` component (not AlertDialog ŌĆö no destructive pattern)
- Title: "Suggest a merge" (16px, semibold)
- Body: "Your adaptation notes, rating, and feedback will be shared with the reviewer automatically." (13px, muted)
- Single `Textarea`: label "Why should this be merged back?" ŌĆö required, min-height 80px
- Footer: "Cancel" (ghost) + "Submit suggestion" (primary `#4287FF`)
- On submit: optimistic UI ŌĆö button shows spinner via `useTransition`, dialog closes on success, toast: "Merge suggestion submitted"

**Pending state (suggestion submitted, awaiting review):**
- Button replaced by status badge: amber `#FFB852` background (10% opacity) + amber text, icon `Clock` 12px
- Badge copy: "Pending Review"
- No interaction on badge ŌĆö read-only

**Merged state (admin approved):**
- Status badge: teal `#65CFB2` background (10% opacity) + teal text, icon `Check` 12px
- Badge copy: "Merged"

**Declined state (admin declined):**
- Status badge: red `#E3392A` background (10% opacity) + red text, icon `X` 12px
- Badge copy: "Declined"
- Below badge: admin's decline reason rendered at 13px muted-foreground, italic
- "Revise & resubmit" link (13px, `#4287FF`) below reason ŌĆö re-enables the suggestion dialog with pre-filled previous merge note

### Review Queue Page (`/review`)

**Layout:** Single-column list, max-w consistent with library page. Page heading: "Review Queue" (20px, semibold). Filter tabs row directly below heading.

**Filter tabs:** "Pending" | "Approved" | "Declined" | "All" ŌĆö `Tabs` component, active tab underlined with `#4287FF`. Default: Pending.

**Review queue card anatomy (vertical stack, py-16px px-16px, bg-card, border-border, rounded-md):**
- Row 1: prompt title (16px, semibold) + status badge (right-aligned)
- Row 2: "Suggested by {name} ┬Ę {engagement name} ┬Ę {relative time}" (13px, muted)
- Row 3: star rating display (read-only StarRating, showLabel={false}) + effectiveness label "13px muted"
- Row 4: merge note preview ŌĆö truncated to 2 lines, 13px, text-foreground, italic. Tooltip on hover shows full text.
- Card hover: `border-[#4287FF]` transition (matches library card hover pattern from Phase 2)
- Card click: navigates to `/review/[suggestionId]`

**Empty state (Pending tab):**
- Heading: "No pending merge suggestions"
- Body: "When consultants suggest merging their improvements, they'll appear here for review."
- Icon: `GitMerge` (lucide, 32px, muted)
- No CTA ŌĆö admin cannot create merge suggestions

**Loading state:** 3 skeleton cards at card height, staggered opacity

**Sidebar nav badge:** `PendingCountBadge` renders "Review Queue (N)" where N = count of pending suggestions. Fetched server-side at layout render. N=0 shows no badge ŌĆö item reads "Review Queue" only.

### Review Detail Page (`/review/[suggestionId]`)

**Layout:** Two-column, identical proportions to `ForkDetailClient` ŌĆö `flex gap-8 items-start`. Left: `flex-1 min-w-0`. Right: `w-[280px] shrink-0`.

**Page header (above columns):**
- Back link: `ArrowLeft` icon + "Back to Review Queue" ŌĆö same pattern as fork detail
- Prompt title: 20px, semibold

**Left column ŌĆö Diff + optional content editor:**
- Section heading: "Changes" (16px, semibold, border-b border-border pb-2 mb-4)
- `DiffViewer` component rendered directly ŌĆö `leftTitle="Library (current)"`, `rightTitle="Fork (adapted)"`
- Below diff: collapsible section "Edit before approving" (collapsed by default)
  - Trigger: "Edit content" button ŌĆö `variant="ghost"`, `ChevronDown` icon, 14px
  - Expanded: `Textarea` pre-filled with adapted content (monospace 13px), full height auto-resize
  - Label: "This content will replace the library prompt on approval." (13px, muted)

**Right sidebar ŌĆö Context + actions (`ReviewSidebar`):**
Section structure mirrors ForkSidebar (py-4 + border-t pattern):

- Section 1 ŌĆö Who / Where (no border-t, first section)
  - Label: "Suggested by" (13px, muted)
  - Value: consultant name (14px)
  - Label: "Engagement" (13px, muted)
  - Value: engagement name with external link icon ŌåÆ `/engagements/[id]`

- Section 2 ŌĆö Effectiveness
  - `StarRating` component, `showLabel={true}`, read-only (no `onRate` prop)

- Section 3 ŌĆö Issue Tags
  - Read-only `IssueTagGroup` display (or flat badge list if no onToggle wired)

- Section 4 ŌĆö Feedback Notes
  - Label: "Consultant feedback" (13px, muted)
  - Value: feedback text or "ŌĆö" if none (13px)

- Section 5 ŌĆö Adaptation Notes
  - Label: "Adaptation notes" (13px, muted)
  - Value: adaptation text or "ŌĆö" if none (13px)

- Section 6 ŌĆö Merge Note
  - Label: "Merge note" (13px, muted)
  - Value: merge note text (13px, italic)

- Section 7 ŌĆö Actions
  - "Approve & Merge" button: `variant="default"` (primary `#4287FF`), full width, icon `Check`
  - `DeclineReasonForm` below (see interaction below)

**Decline inline flow (`DeclineReasonForm`):**
- Initially: "Decline" button, `variant="outline"` full width, icon `X`
- On click: button replaced by inline form ŌĆö no modal
  - `Textarea`: placeholder "Why are you declining this? The consultant will see your reason." min-height 64px
  - Two buttons below: "Cancel" (ghost, collapses form) + "Confirm Decline" (destructive `#E3392A`, requires non-empty textarea)
- On confirm: server action, toast "Merge suggestion declined", redirect to `/review`

**Approve flow:**
- "Approve & Merge" click ŌåÆ `useTransition` spinner on button, disabled state
- On success: toast "Prompt updated ŌĆö version bumped", redirect to `/review`
- On error: toast error "Couldn't approve merge. Try again."
