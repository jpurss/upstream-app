## Color

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `#09090b` (zinc-950) | Page backgrounds, app shell background |
| Secondary (30%) | `#18181b` (zinc-900) | Sidebar background, cards, input fields, modal overlays |
| Accent (10%) | `#4287FF` | Listed elements below only |
| Destructive | `#ef4444` (red-500) | Destructive actions only (none in Phase 1) |
| Muted text | `#71717a` (zinc-500) | Disabled nav items, secondary metadata, "No signup required" subtext |
| Border | `#27272a` (zinc-800) | Card borders, dividers, sidebar separator |

Accent (`#4287FF` — Human Agency brand blue) reserved for:
- Primary CTA buttons ("Explore the Demo", "Explore as Consultant", "Explore as Admin", "Sign In")
- Active/selected sidebar navigation item indicator
- Focus ring on interactive elements
- Demo mode banner background tint (at 15% opacity over zinc-950)

Accent is NOT applied to: body text, labels, icons in non-selected states, card borders, form fields.

**Source:** D-16 (CONTEXT.md, REQUIREMENTS.md UI-01). Zinc dark palette from shadcn zinc preset. Muted/border values from zinc scale defaults.

## Page-Level Layout Specifications

### Login Page (`/login`)

Layout: Two-column on desktop (min 1280px). Left column: 60% width, product context (name, tagline, 2-3 feature highlights). Right column: 40% width, authentication panel.

Authentication panel (right column) vertical order:
1. Product wordmark (20px semibold, Geist Sans, white)
2. Vertical gap: 32px
3. "Explore the Demo" section heading (13px semibold, zinc-400, uppercase tracking-wide)
4. "Explore as Consultant" button — full-width, accent fill (#4287FF), 16px semibold
5. "Explore as Admin" button — full-width, accent outline (border #4287FF, text #4287FF), 16px semibold
6. "No signup required" subtext — 13px, zinc-500, centered
7. Separator with "or" label — zinc-800 border, "or" in 13px zinc-500
8. Email input — label "Email", 14px, zinc-900 bg, zinc-800 border
9. Password input — label "Password", same as email
10. "Sign In" button — full-width, accent fill, 14px semibold
11. "Don't have an account? Sign up" — 13px zinc-500, "Sign up" in #4287FF

Left column feature highlights: 3 items with Lucide icon + heading (16px semibold) + body (14px zinc-400). Topics: Prompt Library, Fork & Adapt, Merge & Improve.

**Source:** D-01, D-02, D-04 (CONTEXT.md), SPECIFICS section (CONTEXT.md).

### App Shell (post-login)

Layout: Sidebar (left) + main content area (right). Sidebar expanded width: 240px. Sidebar collapsed width: 48px. Transition: 200ms ease-out.

Sidebar structure (top to bottom):
1. Logo/wordmark area (16px semibold "Upstream", collapsed: "U" monogram)
2. Navigation items — Library (active), Engagements (disabled), Demand Board (disabled), Review Queue (admin only, disabled), Dashboard (admin only, disabled)
3. Bottom section: User avatar + name + role badge, logout button

Disabled nav items: text color zinc-600, no hover state, `cursor-not-allowed`, no click handler. Active item: text white, #4287FF left border indicator (2px), zinc-800 background.

Demo banner: When session is anonymous — renders as 32px fixed strip at top of viewport (above sidebar and content area), full-width. Contains: "Demo mode — [Consultant / Admin] view. Sign up to save your work." with a "Sign Up" text link in #4287FF.

Library page placeholder (Phase 1 only): Centered empty state — "Prompt Library" heading (20px semibold), "18 prompts loaded and ready — full library browsing comes in Phase 2." body (14px zinc-400). No interactive elements.

**Source:** D-05, D-06, D-07, D-03 (CONTEXT.md). shadcn Sidebar collapsible="icon" pattern (RESEARCH.md).

## Interaction States

| Element | States |
|---------|--------|
| Buttons (accent fill) | default → hover (brightness +10%, no scale) → active (brightness -5%) → loading (spinner, disabled) → focus (2px #4287FF ring, 2px offset) |
| Buttons (outline) | default → hover (bg zinc-900) → active (bg zinc-950) → focus (2px #4287FF ring) |
| Nav items (enabled) | default (zinc-400) → hover (zinc-100 text, zinc-800/50 bg) → active (white text, zinc-800 bg, #4287FF left border) |
| Nav items (disabled) | zinc-600 text, no hover, cursor-not-allowed |
| Form inputs | default (zinc-800 border) → focus (zinc-600 border, #4287FF ring) → error (red-500 border + error message below) → filled (zinc-700 border) |
| Sidebar toggle | icon rotates 180deg on collapse, 150ms ease |
| Demo buttons | same as primary button states — no special treatment beyond label |

**Source:** Linear/Raycast aesthetic (UI-02, REQUIREMENTS.md). Standard interaction patterns for zinc dark shadcn preset.

## Accessibility Baseline

- All interactive elements must be keyboard-reachable (Tab order follows visual order)
- Focus rings: 2px solid #4287FF, 2px offset — never hidden
- Disabled nav items: `aria-disabled="true"`, `tabindex="-1"`
- Demo banner: `role="banner"`, `aria-label="Demo mode indicator"`
- Color contrast: Body text (zinc-100 on zinc-950) = 18.5:1. Passes WCAG AA and AAA.
- Minimum touch target: 44px height for all buttons (desktop-only product but baseline enforced)
- Form error messages: `aria-live="polite"`, associated via `aria-describedby`
