## Spacing Scale

Declared values (multiples of 4 only). Source: .impeccable.md — unchanged from Phase 2.

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, badge internal padding, status dot margins |
| sm | 8px | List item padding, form field gaps, issue tag spacing |
| md | 16px | Card padding, nav item padding, sidebar section gaps |
| lg | 24px | Section padding, form group spacing, modal header padding |
| xl | 32px | Layout section gaps, two-column gap in fork detail |
| 2xl | 48px | Major section breaks, workspace header bottom margin |
| 3xl | 64px | Page-level outer padding |

Exceptions:
- Engagement card grid gap: 16px (md) between cards — consistent with library grid
- Fork card grid gap: 16px (md) between fork cards
- Touch targets (clickable cards, star rating buttons): minimum 36px height
- Star rating row height: 32px — stars are 20px with 8px tap padding
- Prompt picker modal: full-screen overlay, inner content max-width 800px
- Engagement status dropdown trigger: 32px height (compact inline control)
- Autosave indicator text: inline in page header, no dedicated height allocation

## Typography

Source: .impeccable.md. Two weights only — 400 (regular) and 600 (semibold). Identical to Phase 2.

| Role | Size | Weight | Line Height | Font | Usage in Phase 3 |
|------|------|--------|-------------|------|-----------------|
| Display | 20px | 600 | 1.2 | Geist Sans | Engagement name as H1 in workspace header, page title "Engagements" |
| Heading | 16px | 600 | 1.3 | Geist Sans | Engagement card titles, fork card prompt titles, fork detail section headers, modal titles |
| Body | 14px | 400 | 1.5 | Geist Sans | Default UI text, form labels, client name, industry badge labels, adaptation notes |
| Label | 13px | 400 | 1.4 | Geist Sans | Status indicators, fork count, relative dates, issue tag text, avg effectiveness metadata |
| Prompt content | 13px | 400 | 1.6 | Geist Mono | Fork adapted content in Write tab and Preview tab — never UI chrome |

Notes:
- No other font sizes permitted in this phase.
- Engagement card client name uses body (14px) at weight 400, muted foreground.
- Fork card status indicators (customization/adaptation) use label (13px) with semantic color dots.
- Prompt content headings inside Preview/Diff tabs use Geist Sans: H1 at 18px/600 with border-b, H2 at 16px/600, H3 at 14px/600. Body content remains Geist Mono 13px.
- Issue tag badges use label (13px) weight 400.

## Color

Source: .impeccable.md / globals.css. CSS variable names confirmed from codebase scan. Identical to Phase 2 plus new semantic usages.

| Role | Value | CSS Variable | Usage |
|------|-------|-------------|-------|
| Background (60%) | `#09090b` zinc-950 | `--background` | Page backgrounds, app shell, modal overlays |
| Surface (30%) | `#18181b` zinc-900 | `--card` | Engagement cards, fork cards, sidebar, modal body, picker modal inner surface |
| Accent (10%) | `#4287FF` | `--primary` / `--ring` | Reserved for specific elements listed below |
| Destructive | `#E3392A` | `--destructive` | Delete/abandon actions (none in v1 of this phase) |
| Warning | `#FFB852` | `--warning` | Paused engagement status dot, pending/attention states |
| Success | `#65CFB2` | `--success` | Active engagement status dot, merged/healthy states |
| Completed | `#71717a` zinc-500 | `--muted-foreground` | Completed engagement status dot — muted, not actionable |
| Text primary | `oklch(0.985 0 0)` | `--foreground` | All primary content |
| Text muted | `#71717a` zinc-500 | `--muted-foreground` | Client name, dates, fork count, metadata, placeholder text |
| Border | `#27272a` zinc-800 | `--border` | Card borders, dividers, sidebar separators, input borders |

**Accent (`#4287FF`) reserved for:**
1. "New Engagement" primary CTA button (hero state in empty page, secondary in populated page header)
2. "+ Fork a Prompt" button in workspace header
3. "Fork" confirm button in fork-to-engagement dialog
4. Active star in effectiveness rating when filled (index ≤ user-selected rating) — use `#FFB852` (amber) for star fill, NOT blue
5. Focus ring on all inputs, textareas, and interactive controls (`--ring`)
6. Active nav item — "Engagements" when on any `/engagements/` route
7. Active/selected issue tag badge border and background tint (`bg-primary/10 border-primary/30`)
8. "Suggest Merge" button — reserved but deferred to Phase 4

Accent is NOT used for: industry badges, status dots, fork count chips, relative dates, or disabled states.

**Star rating color:** Stars use `#FFB852` (amber/warning) for filled state, not accent blue. Filled when the selected rating index is ≤ the star's position. Outline-only for unselected. This is a quality/effectiveness signal — amber, not brand blue.

## Interaction Patterns

Source: CONTEXT.md decisions D-01 through D-25.

### Engagement List Page — Card Grid (D-02, D-03, D-04)

- Layout: card grid, same responsive breakpoints as library grid — 3 columns at `lg`, 4 columns at `xl`
- Grid gap: 16px (md)
- Engagement card contents:
  - Engagement name — heading size (16px/600), full width
  - Client name — body size (14px/400), muted foreground, below name
  - Industry badge — `<Badge>` component, muted surface variant, label size (13px)
  - Status indicator — colored dot (8px circle) + status label text at label size (13px): Active = teal `#65CFB2`, Paused = amber `#FFB852`, Completed = muted `#71717a`
  - Fork count — label size (13px), muted, lucide `GitFork` icon at 12px
  - Last activity — relative date ("3 days ago"), label size (13px), muted foreground
- Card hover: same pattern as library — `border-primary/30` on hover, no elevation lift
- Whole card is a `<Link>` to `/engagements/[id]`
- "New Engagement" button: primary variant in page header, right-aligned. Text: "New Engagement"

### Engagement Empty State (D-04)

- Centered vertically in viewport remaining height
- lucide `Briefcase` icon at 40px, muted foreground, above heading
- Heading: "Create your first engagement" — heading size (16px/600)
- Body: "Fork prompts into client workspaces, adapt them, and rate what works." — body size (14px), muted foreground, max-width 320px, centered
- CTA: "New Engagement" primary button, centered below body text
- Subtle link below CTA: "or Browse the Library →" — label size (13px), muted foreground, hover foreground, no underline

### Create Engagement — Modal Dialog (D-05, D-06)

- Trigger: "New Engagement" button on list page
- Component: `<Dialog>` with header, 3 form fields, footer actions
- Fields:
  1. Name — text `<Input>`, required, label "Engagement Name", placeholder "e.g. Accenture AI Strategy Q2"
  2. Client Name — text `<Input>`, required, label "Client Name", placeholder "e.g. Accenture"
  3. Industry — `<Select>` dropdown, label "Industry", required
- Footer: "Create Engagement" primary button (left of cancel) + "Cancel" ghost button
- On success: optional second step shows prompt picker (D-06) — "Add prompts to get started" heading, searchable mini library card grid, "Skip for now" ghost link, "Fork Selected (N)" primary button
- Error: inline field validation — "This field is required." at 13px destructive below each empty required field

### Industry Dropdown Values (discretion)

Active, Paused, Completed are engagement status values. Industry is a separate field. Values (alphabetical):
Financial Services, Government, Healthcare, Legal, Manufacturing, Media, Professional Services, Retail, Technology, Telecommunications, Other

### Engagement Status Change (D-08)

- Trigger: status badge/dropdown in workspace header — click to reveal dropdown
- Component: `<Select>` inline in header, current status as trigger label with colored dot
- Options: Active, Paused, Completed
- Completed selection: triggers `<AlertDialog>` confirmation before saving
  - See Copywriting section for dialog copy
- Autosave on selection (for Active/Paused — no confirmation needed)

### Engagement Workspace — Header (D-07, D-09)

- H1: engagement name at display size (20px/600)
- Subline: client name + industry badge, body size (14px), muted
- Right side: status `<Select>` with colored dot, fork count chip (label size), avg effectiveness display ("4.2 avg" with lucide `Star` icon at 12px)
- "+ Fork a Prompt" button: primary variant, right-aligned in header area. Icon: lucide `Plus` at 16px

### Fork Cards (D-10, D-11)

Fork cards are work-status trackers, NOT prompt previews.

- Card surface: `--card` (#18181b), same border/radius as library cards
- Card contents:
  - Prompt title — heading size (16px/600)
  - Customization status indicator — label size (13px): "Template ready" (muted) if `{{variables}}` remain unfilled, "Customized" (teal `#65CFB2` dot) if variables filled, or no indicator if no variables
  - Adaptation status indicator — label size (13px): "Adapted" (blue `#4287FF/70` dot) if adapted_content differs from original_content beyond template fills, "Original" (muted dot) if unchanged
  - Last edited — relative date, label size (13px), muted foreground, lucide `Clock` icon at 12px
- Card hover: `border-primary/30` transition — same as library cards
- Whole card is a `<Link>` to `/engagements/[id]/forks/[forkId]`

### Fork Creation — Three Entry Points (D-12, D-13, D-14, D-15, D-16)

**Entry Point 1: Prompt detail page sidebar (D-12)**
- Button: "Fork to Engagement" — secondary variant, full width, below existing sidebar actions
- Opens `<Dialog>`: title "Fork to Engagement", engagement list as radio items
- Each item: engagement name + client name subline + status dot
- "Already forked" state: item grayed out with label "Already in this engagement" — not selectable
- Footer: "Fork" primary button + "Cancel" ghost button
- If no active engagements: dialog shows "No active engagements" with inline "Create one" link that opens create engagement modal

**Entry Point 2: Workspace "+ Fork a Prompt" button (D-13)**
- Full-screen modal (Dialog at max-width 800px with max-height 80vh overflow scroll)
- Header: "Fork a Prompt" title, search `<Input>` below, filter subset (Category, Capability Type as `<Select>` dropdowns)
- Body: mini library card grid — reuses `<LibraryGrid>` filtering logic
- Already-forked prompts: rendered with "Already forked" badge overlay on card, pointer-events none
- Multiple selection: checkboxes on cards, selection count shows in footer "3 selected"
- Footer: "Fork Selected (N)" primary button + "Cancel" ghost button
- "Fork Selected" disabled when no prompts selected

**Entry Point 3: Post-creation prompt picker (D-14)**
- Same as Entry Point 2 UI but rendered as a step in the create engagement modal flow
- Footer: "Fork Selected (N)" primary button + "Skip for now" ghost link (no confirmation)

### Fork Detail — Page Layout (D-17, D-18, D-19, D-20)

- Route: `/engagements/[id]/forks/[forkId]`
- Two-column layout: wide editor column (`flex-1`, min 0) + narrow metadata sidebar (280px fixed)
- Column gap: 32px (xl)
- Page header: back link "← [Engagement Name]" (body size, muted, hover foreground) + autosave indicator (right-aligned, label size 13px, muted)
- Autosave indicator states: blank (no unsaved changes) → "Saving..." (during debounce/request) → "Saved" (2s after confirm, then returns to blank)
- Autosave debounce: 1.5 seconds after last keystroke

### Fork Editor — Write/Preview/Diff Tabs (D-19, D-24, D-25)

- Component: `<Tabs>` with three tabs: "Write", "Preview", "Diff"
- Write tab: `<Textarea>` in `font-mono` (Geist Mono), full height, no resize handle, `min-h-[400px]`
- Preview tab: react-markdown render with `remark-gfm` — same prose rendering as library detail page
- Diff tab: side-by-side diff component
  - Left column header: "Original" — label size, muted foreground
  - Right column header: "Adapted" — label size, muted foreground
  - Diff library: `react-diff-viewer-continued` — confirm dark mode theming before implementation (see STATE.md blocker)
  - If original === adapted: show empty state "No changes yet. Edit in Write mode to see a diff."

### Fork Sidebar — Metadata and Feedback (D-21, D-22, D-23)

Sidebar sections top to bottom:

1. **Effectiveness Rating** — label "Rate effectiveness", 5 star icons (`lucide Star`, 20px each, 4px gap), amber fill on selected and below, outline on unselected. Click any star to set rating. Autosave on click. `aria-label="Rate effectiveness N out of 5"` on each star.

2. **Issue Tags** — label "Issue tags", clickable badge toggles. Tags: "Hallucination", "Too verbose", "Wrong format", "Model degradation", "Needs context". Active tag: `bg-primary/10 border-primary/30 text-primary`. Inactive tag: `bg-muted border-border text-muted-foreground`. Autosave on toggle.

3. **Feedback Notes** — label "Feedback notes", freeform `<Textarea>` (Geist Sans, not Mono), `min-h-[80px]`, placeholder "What worked, what didn't...". Autosave on change (same 1.5s debounce).

4. **Client Context** — `<Checkbox>` + label "Contains client-specific context". Autosave on toggle.

5. **Adaptation Notes** — label "Adaptation notes", `<Textarea>` (Geist Sans), `min-h-[80px]`, placeholder "What you changed and why...". Autosave on change.

6. **Source Prompt** — label "Source prompt", link to `/library/[promptId]` with prompt title. lucide `ExternalLink` icon at 12px. Muted text, hover foreground.

7. **Forked** — label "Forked", date string, muted text, label size.

Sidebar dividers: `border-t border-border` between each section, `py-4` (16px) on each section.

### Loading States

- Engagement list skeleton: 8 `<Skeleton>` cards matching engagement card dimensions, `animate-pulse`
- Engagement workspace skeleton: workspace header skeleton (2 lines) + 6 fork card skeletons
- Fork detail skeleton: editor area = single large skeleton block; sidebar = 5 skeleton section blocks
- Skeleton card dimensions: 100% width, 120px height for engagement cards, 96px for fork cards

### Toast Notifications

- Fork created (single): "Forked to [Engagement Name]" — sonner toast, 3s duration, no action
- Fork created (multiple): "3 prompts forked to [Engagement Name]" — sonner toast
- Engagement created: "Engagement created" — sonner toast, 3s
- Autosave errors: "Couldn't save changes. Retrying..." — sonner toast, destructive variant, auto-dismiss after 5s

**Voice rules (from .impeccable.md):**
- Labels describe what things do — not what they aspire to be
- No exclamation marks
- No "successfully" prefix — state the outcome: "Saved", not "Successfully saved!"
- Confirmation dialogs state consequence before action
- Dynamic values in brackets, e.g. "[Engagement Name]"
