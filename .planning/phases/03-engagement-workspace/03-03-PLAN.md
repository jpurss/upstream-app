**Step 1 — Create `components/engagements/fork-card.tsx` (client component):**

Per D-10, D-11, and UI-SPEC fork card spec:

Fork cards are work-status trackers, NOT prompt previews. Library metadata is irrelevant once forked.

Card contents:
- Prompt title — heading size (16px/600)
- Customization status indicator — label size (13px):
  - If `adapted_content` contains `{{...}}` template variables: "Template ready" (muted text)
  - If no template variables remain AND `adapted_content` differs from `original_content`: "Customized" with teal dot `#65CFB2`
  - If no template variables in original: no customization indicator shown
- Adaptation status indicator — label size (13px):
  - If `adapted_content !== original_content` (beyond template fills): "Adapted" with blue dot `#4287FF` at 70% opacity
  - If unchanged: "Original" with muted dot
- Last edited — relative date (reuse the `getRelativeTime` helper), 13px muted, lucide `Clock` icon at 12px

Card surface: `--card` (#18181b), same border/radius as library cards. Hover: `border-primary/30` transition. Whole card is `<Link>` to `/engagements/[engagementId]/forks/[forkId]`.

Status detection logic:
```typescript
function getForkStatusIndicators(fork: ForkedPromptWithTitle) {
  const hasTemplateVars = /\{\{[^}]+\}\}/.test(fork.adapted_content ?? '')
  const isContentChanged = fork.adapted_content !== fork.original_content
  return { hasTemplateVars, isContentChanged }
}
```

Props: `{ fork: ForkedPromptWithTitle; engagementId: string }`

**Step 3 — Create `app/(app)/engagements/[id]/page.tsx` (server component):**

Async server component:
1. Await `params` to get `id`: `const { id } = await params`
2. Create Supabase client, get authenticated user
3. Fetch engagement: `const engagement = await fetchEngagementById(id)`
4. If not found: `notFound()`
5. Fetch forks: `const forks = await fetchForksByEngagement(id)`
6. Compute stats: `forkCount = forks.length`, `avgEffectiveness = ...` (average of non-null effectiveness_rating values)
7. Render:
   - `<WorkspaceHeader>` with engagement data, fork count, avg effectiveness
   - `<ForkGrid>` with forks data and engagement ID

Page params type follows Next.js 16 pattern: `{ params: Promise<{ id: string }> }`
