  <files>proxy.ts, lib/supabase/middleware.ts</files>
  <action>
**Root cause:** proxy.ts exists and is correctly named for Next.js 16 (proxy.ts with `export async function proxy` is the Next.js 16 convention — middleware.ts is deprecated). It currently only handles session refresh and unauthenticated-to-login redirect. It does NOT handle role-based routing. When a consultant logs in and gets redirected to /engagements via server action, the (app) layout's force-dynamic async operations briefly render the library loading skeleton before the engagements page resolves.

**Fix approach:** Add role-based redirect logic in `lib/supabase/middleware.ts` (where the user object is already available from `getUser()`), so the proxy intercepts default-route navigations and redirects consultants at the edge before any server components load.

**In `lib/supabase/middleware.ts` — `updateSession()`:**

After the existing `getUser()` call and the unauthenticated redirect block, add role-based routing:

```typescript
// Role-based routing: redirect consultants away from /library to /engagements
// This eliminates the library skeleton flash during post-login navigation
if (user) {
  const pathname = request.nextUrl.pathname
  const role = user.app_metadata?.role as string | undefined
  const isAnonymous = user.is_anonymous ?? false
  const demoRole = isAnonymous
    ? (user.user_metadata?.demo_role as string | undefined) ?? 'consultant'
    : undefined
  const effectiveRole = role ?? demoRole ?? 'consultant'

  // Consultants landing on /library should be redirected to /engagements
  // (Library is still accessible via sidebar nav, but not the default landing)
  if (effectiveRole !== 'admin' && pathname === '/library') {
    const url = request.nextUrl.clone()
    url.pathname = '/engagements'
    return NextResponse.redirect(url)
  }
}
```

**Key design decisions:**
- Only redirect exact `/library` path, NOT `/library/[promptId]` — consultants must still be able to view individual prompt detail pages from the library
- Use the same effectiveRole derivation as `app/(app)/layout.tsx` (app_metadata.role -> user_metadata.demo_role fallback)
- Redirect returns immediately, so the (app) layout never loads for this navigation
- Do NOT redirect `/library` for admins — that's their landing page

**In `proxy.ts`:** No changes needed. It already imports and calls `updateSession()` correctly with the right matcher. The file uses `export async function proxy` which is the correct Next.js 16 convention (middleware.ts was renamed to proxy.ts in Next.js 16).
