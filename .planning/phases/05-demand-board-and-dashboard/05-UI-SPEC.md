## Spacing Scale

Declared values (multiples of 4 only):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, badge internal padding, upvote arrow-to-count gap |
| sm | 8px | List item padding, form field gaps, card metadata row gaps |
| md | 16px | Card padding, dialog padding, form label-to-input gap |
| lg | 24px | Section padding, page header to content gap, filter tab row padding |
| xl | 32px | Layout section gaps, dashboard metric card group gap |
| 2xl | 48px | Major section breaks between dashboard zones |
| 3xl | 64px | Page-level outer padding |

Exceptions:
- Upvote column width: 48px (accommodates arrow + count + comfortable tap zone)
- Request card touch target minimum: 44px height
- Dashboard metric card: minimum 120px height (enough for label + large number + trend)
- Chart container heights: line chart 300px, bar chart 300px — fixed to prevent layout shift
- Filter tab row: 40px height (matches established pattern from review queue, Phase 4)

Source: `.impeccable.md` SS Spacing; Phase 4 UI-SPEC established exceptions

## Interaction Patterns

Source: `05-CONTEXT.md` decisions D-01 through D-17.

### Request Cards (D-01, D-02)

**Card anatomy (flex row, `bg-card`, `border border-border`, `rounded-md`, `p-4`):**

```
+--48px--+------------------------------------+
|   ^    | [Title] (20px semibold)            |
|  [N]   | [Description truncated 2 lines]    |
|        | [Category badge] [Urgency badge]   |
|        | [Submitter name] · [relative time] |
|        | [Resolved: → Prompt Name (link)]   |
+--------+------------------------------------+
```

- **Upvote column (left, 48px fixed width):** vertical stack — `ArrowUp` icon (lucide, 16px), count (13px, weight 600, foreground). Arrow is filled blue (`text-primary fill-primary`) when current user has upvoted, outline-only when not. One click toggles state. On click: optimistic update (immediate visual change) + server action.
- **Content area (flex-1):** title (20px, 600), description (`line-clamp-2`, 15px), badge row (category + urgency), metadata row (submitter + relative time — 13px muted). All at `gap-1` vertical stacking.
- Resolved cards only: below metadata row, show "→ {Prompt Title}" as a link (13px, `text-[#65CFB2]`, hover underline).
- Card hover: `border-[#4287FF]/30` transition (matches library card pattern). Cursor is `default` — cards are NOT clickable as a whole. Only the upvote arrow is interactive for consultants.
- Declined cards: `opacity-60` on the full card (matches deprecated prompt pattern from Phase 2).

### Upvote Interaction (D-01, D-03)

- The upvote button is the ONLY clickable element for consultant users (other than "New Request").
- On click (not yet upvoted): arrow fills blue, count increments by 1. `useTransition` — optimistic update first, then server action. On error: revert, toast "Could not save upvote. Try again."
- On click (already upvoted): arrow returns to outline, count decrements by 1. Same optimistic pattern.
- Anonymous demo user: upvote button is enabled (demo session = consultant role with demo user ID).
- Upvote button `aria-label`: "Upvote this request" or "Remove upvote" based on state.

### Filter Tabs (D-03)

- Five tabs: "Open" | "Planned" | "Resolved" | "Declined" | "All"
- Default active: "Open"
- Tab component: `Tabs` from shadcn. Active tab underlined with `#4287FF` (matches review queue pattern, Phase 4).
- URL state: `router.push` with `?status=open|planned|resolved|declined|all` param — consistent with Phase 4 review queue pattern (not nuqs for tab state).
- Tab labels include count badge when non-zero: "Open (12)", "Planned (3)", etc. Count badge: 12px, `bg-muted rounded-full px-1.5`, muted-foreground text. Badge on "All" omitted (redundant).

### Sort Controls (D-03)

- Sort options: "Most upvoted" (default) | "Newest first" | "Urgent first"
- Rendered as a `Select` dropdown, right-aligned in the same row as the filter tabs.
- Sort selection persisted in URL via `router.push` with `?sort=upvotes|newest|urgent`.
- Label prefix: "Sort:" at 13px muted, inline before the Select trigger.

### New Request Dialog (D-04)

- Trigger: "New Request" button, right-aligned in page header, primary variant, all authenticated users.
- `Dialog` component (not AlertDialog — not a destructive action).
- Dialog title: "New prompt request" (20px, semibold).
- Fields (all required):
  1. Title: `Input`, label "What prompt do you need?", placeholder "e.g. Competitor analysis framework prompt"
  2. Description: `Textarea`, label "Describe the use case", min-height 80px, placeholder "When would you use this? What should it help you do?"
  3. Category: `Select`, label "Category", same 6 options as library (Discovery, Solution Design, Build, Enablement, Delivery, Internal Ops)
  4. Urgency: `Select`, label "Urgency", options: "Nice to have", "Medium", "Urgent"
- Footer: "Discard request" (ghost) + "Submit Request" (primary `#4287FF`)
- On submit: `useTransition` spinner on button, dialog closes on success, toast: "Prompt request submitted"
- Client-side validation: title and description required before button activates (HTML5 required attributes + button disabled state).

### Admin Controls on Cards (D-05, D-06, D-07, D-08)

Admin controls appear directly on the card — no separate request detail page. Consultant users see no admin controls (conditional render in JSX, not disabled/hidden CSS).

**Open request card — admin view:** Two additional buttons in the top-right of the card content area:
- "Mark Planned" — `variant="outline"` ghost, 13px, icon `Clock` (size-3). On click: server action, card status badge updates to Planned (amber).
- "Resolve" — `variant="outline"` ghost, 13px, icon `Link` (size-3). On click: opens Resolve Dialog.
- "Decline" — `variant="ghost"` destructive text color, 13px, icon `X` (size-3). On click: expands inline decline form below the card content.

**Planned request card — admin view:** Replaces "Mark Planned" with "Revert to Open" (ghost, 13px).

**Resolve Dialog (D-06):**
- `Dialog` component.
- Dialog title: "Resolve this request"
- Dialog body: "Link this request to the prompt that addresses it." (13px, muted)
- Search field: `Input` with `Search` icon prefix, label "Find prompt", placeholder "Type to search active library prompts..."
- Autocomplete results: dropdown list (`bg-popover border-border rounded-md shadow-md`) below the input — each row shows prompt title (15px) + category badge (13px). Max 5 results visible (scroll for more).
- Select a prompt: input fills with prompt title, row highlights with `bg-accent`.
- Footer: "Cancel" (ghost) + "Confirm Resolve" (primary `#4287FF`, disabled until a prompt is selected)
- On success: dialog closes, card status updates to Resolved (teal), linked prompt name appears in card.

**Inline Decline Form (D-07):**
Expands below the card content area — same inline expansion pattern as merge decline from Phase 4:

```
[Textarea: "Why are you declining this request?..."]   [min-height 64px]
[Discard (ghost)]  [Confirm Decline (destructive, disabled until non-empty)]
```

- `Confirm Decline` button activates only when textarea has content.
- "Discard" collapses the form back to the read-only card.
- On success: form collapses, card status updates to Declined (red), card dims to 60% opacity.

### Dashboard Layout (D-09, D-10, D-11)

**Three-zone vertical stack:**

```
Zone 1: Metric Cards (3 cards across the top)
Zone 2: Charts Row (line chart full width, or side-by-side if space allows)
Zone 3: Tables Row (Top 10 + Bottom 10 side-by-side)
```

**Zone 1 — Metric Cards (D-09):**

Three cards in a `grid grid-cols-3 gap-6` layout:

| Card | Metric | Icon |
|------|--------|------|
| Active Prompts | Count of prompts with status = 'active' | `BookOpen` (lucide) |
| Total Checkouts | Count of all forks (all time) | `GitFork` (lucide) |
| Open Items | Pending merge requests + Open prompt requests (two sub-numbers or combined) | `InboxIcon` (lucide) |

Card anatomy (`bg-card border-border rounded-md p-6`):
- Icon: 20px, `text-muted-foreground`, top-left
- Metric number: 28px, weight 600, `text-foreground` — the dominant visual element
- Label: 13px, `text-muted-foreground`, below number
- "Open Items" card shows two sub-lines: "N merge suggestions" + "N prompt requests" (13px each, muted)

**Zone 2 — Charts (D-10):**

*Checkouts Over Time — Line Chart:*
- Full-width container, height 300px, `bg-card border-border rounded-md p-6`
- Section heading: "Prompt Usage Over Time" (20px, 600) above the chart
- Weekly granularity on x-axis (last 12 weeks shown by default)
- Single line: checkout count per week. Line color: `#4287FF` (accent).
- X-axis: week labels (13px, muted). Y-axis: count (13px, muted). No gridlines — use axis only for clean look.
- Tooltip on hover: "{week}: {N} checkouts" — dark card tooltip (`bg-card border-border`).
- Recharts `ResponsiveContainer` + `LineChart` + `Line` + `XAxis` + `YAxis` + `Tooltip`.

*Demand vs Supply — Grouped Bar Chart:*
- Full-width container, height 300px, `bg-card border-border rounded-md p-6`
- Section heading: "Demand vs Supply" (20px, 600) above chart
- Monthly granularity. Two bars per month: "Requests opened" (`#FFB852` amber) and "Resolved" (`#65CFB2` teal).
- Legend below chart: small color swatches + labels (13px, muted).
- X-axis: month labels. Y-axis: count.
- Recharts `BarChart` + `Bar` (two instances) + `Legend` + `XAxis` + `YAxis` + `Tooltip`.

**Zone 3 — Tables (D-10, D-11):**

Two-column `grid grid-cols-2 gap-6`:

*Top 10 Most Used Prompts:*
- Container: `bg-card border-border rounded-md`
- Heading: "Most Used Prompts" (20px, 600), `p-4 border-b border-border`
- Table rows: prompt title (15px) + checkout count right-aligned (13px, muted) + small bar indicator (visual only, proportional to max count, `bg-primary/20 h-1 rounded-full`, inline below title). `py-3 px-4` per row. `border-b border-border` between rows.
- Click on row: navigates to `/library/{id}` (entire row is a link).

*Lowest Rated + Underutilized Prompts (D-11):*
- Container: `bg-card border-border rounded-md`
- Heading: "Needs Attention" (20px, 600), `p-4 border-b border-border`
- Two sub-sections separated by a label row:
  - Sub-section A: "Lowest Rated" (13px, `text-muted-foreground`, non-clickable label row) — bottom 5 by avg_effectiveness, showing title + StarRating (read-only, `showLabel={false}`) + avg score.
  - Sub-section B: "Underutilized" (13px label row) — prompts with 0 forks (all time), showing title + "0 checkouts" in muted.
- Row hover: `bg-accent/40` (matches table row hover convention). Click: navigates to `/library/{id}`.
- If no underutilized prompts: sub-section B shows: "All prompts have been checked out at least once." (13px, muted, italic, `py-3 px-4`).

### Loading States

- Metric cards: 3 skeleton boxes at card dimensions, `animate-pulse`
- Charts: skeleton box at chart container dimensions (full-width, 300px height), `animate-pulse`
- Tables: 5 skeleton rows per table at 48px height, `animate-pulse`
- Demand board: 5 skeleton cards at request card height (~96px), `animate-pulse`
- Skeleton color: `bg-muted` (matches established phase 2/3/4 pattern)

## Recharts Dark Mode Integration

Recharts components use inline `stroke` and `fill` props — they do NOT inherit Tailwind CSS variables automatically. Use these explicit values for dark mode:

| Chart Element | Value |
|---------------|-------|
| Line stroke (primary data) | `#4287FF` |
| Bar fill (requests opened) | `#FFB852` |
| Bar fill (resolved) | `#65CFB2` |
| Axis stroke | `#27272a` (border color) |
| Axis tick text | `#71717a` (muted-foreground) |
| Tooltip background | `#18181b` (card) |
| Tooltip border | `#27272a` (border) |
| Tooltip text | `oklch(0.985 0 0)` (foreground) |
| Grid lines | omit entirely (cleaner without) |

Both chart wrapper components (`UsageLineChart`, `DemandBarChart`) must encapsulate all Recharts props — the parent page passes only typed data, not Recharts configuration. This prevents recharts internals from leaking into page-level code.
