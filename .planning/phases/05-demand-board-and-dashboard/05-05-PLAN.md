Schema tables used by seed data:
```sql
-- engagements
CREATE TABLE engagements (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  name TEXT NOT NULL, client_name TEXT NOT NULL, industry TEXT,
  status TEXT DEFAULT 'active', created_by UUID REFERENCES profiles(id),
  created_at TIMESTAMPTZ DEFAULT NOW(), updated_at TIMESTAMPTZ DEFAULT NOW()
);

-- forked_prompts
CREATE TABLE forked_prompts (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  source_prompt_id UUID REFERENCES prompts(id),
  engagement_id UUID REFERENCES engagements(id),
  forked_by UUID REFERENCES profiles(id),
  adapted_content TEXT, original_content TEXT,
  effectiveness_rating INTEGER, issues TEXT[] DEFAULT '{}',
  feedback_notes TEXT, adaptation_notes TEXT,
  contains_client_context BOOLEAN DEFAULT false,
  merge_status TEXT DEFAULT 'none', merge_suggestion TEXT,
  merge_decline_reason TEXT,
  forked_at TIMESTAMPTZ DEFAULT NOW(), last_used TIMESTAMPTZ
);

-- prompt_requests (with decline_reason from Migration 004)
CREATE TABLE prompt_requests (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  title TEXT NOT NULL, description TEXT, category TEXT,
  urgency TEXT DEFAULT 'nice_to_have', status TEXT DEFAULT 'open',
  decline_reason TEXT,
  requested_by UUID REFERENCES profiles(id),
  resolved_by_prompt UUID REFERENCES prompts(id),
  resolved_at TIMESTAMPTZ, created_at TIMESTAMPTZ DEFAULT NOW()
);

-- request_upvotes
CREATE TABLE request_upvotes (
  request_id UUID REFERENCES prompt_requests(id) ON DELETE CASCADE,
  user_id UUID REFERENCES profiles(id),
  PRIMARY KEY (request_id, user_id)
);
```

    After the existing profile upsert succeeds, add a "claim seed data" step that transfers ownership of placeholder seed records to the current anonymous user. This ensures RLS `engagements_own` and `forked_prompts_own` policies show the seed data as "yours."

    The two placeholder UUIDs are:
    - `DEMO_CONSULTANT_ID = '00000000-0000-0000-0000-000000000001'`
    - `DEMO_ADMIN_ID = '00000000-0000-0000-0000-000000000002'`

    In `signInAsDemo`, after the profile upsert:
    ```typescript
    // Claim seed data — transfer ownership from placeholder demo profiles to the new anonymous user
    const placeholderId = role === 'admin'
      ? '00000000-0000-0000-0000-000000000002'
      : '00000000-0000-0000-0000-000000000001'

    // Only claim if this is NOT the placeholder user itself
    if (user.id !== placeholderId) {
      // Transfer engagements
      await adminClient.from('engagements')
        .update({ created_by: user.id })
        .eq('created_by', placeholderId)

      // Transfer forks
      await adminClient.from('forked_prompts')
        .update({ forked_by: user.id })
        .eq('forked_by', placeholderId)

      // Transfer prompt requests
      await adminClient.from('prompt_requests')
        .update({ requested_by: user.id })
        .eq('requested_by', placeholderId)

      // Transfer upvotes
      await adminClient.from('request_upvotes')
        .update({ user_id: user.id })
        .eq('user_id', placeholderId)
    }
    ```

    This is best-effort (no error handling needed — if claim fails, user sees empty state, which is acceptable). The admin client bypasses RLS so these UPDATEs work regardless of policy constraints.
