## Spacing Scale

Declared values (multiples of 4 only). Source: .impeccable.md

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, badge internal padding, inline tight spacing |
| sm | 8px | List item padding, form field gaps, filter chip spacing |
| md | 16px | Card padding, nav item padding, form label-to-input gap |
| lg | 24px | Section padding, form group spacing, filter bar height |
| xl | 32px | Layout section gaps, page header margin-bottom |
| 2xl | 48px | Major section breaks |
| 3xl | 64px | Page-level outer padding |

Exceptions:
- Filter bar row: 40px height (standard interactive control height)
- Touch targets (buttons, clickable cards): minimum 36px height per density principle
- Card grid gap: 16px (md) between cards
- List row height: 52px for density — enough for two lines of text without crowding

## Typography

Source: .impeccable.md. Two weights only — 400 (regular) and 600 (semibold).

| Role | Size | Weight | Line Height | Font | Usage in Phase 2 |
|------|------|--------|-------------|------|------------------|
| Display | 20px | 600 | 1.2 | Geist Sans | Page title "Prompt Library", form page titles |
| Heading | 16px | 600 | 1.3 | Geist Sans | Card titles, detail page section headers, form field group headers |
| Body | 14px | 400 | 1.5 | Geist Sans | Default UI text, filter labels, card description truncated, form labels, nav items |
| Label | 13px | 400 | 1.4 | Geist Sans | Badges (category, model), metadata (checkout count, rating count, timestamps) |
| Prompt content | 13px | 400 | 1.6 | Geist Mono | Prompt body text on detail page and Write/Preview tabs — never UI chrome |

Notes:
- No other font sizes permitted in this phase.
- Truncated card descriptions use CSS `line-clamp-1` at body (14px) size.
- Muted metadata (`#71717a`) uses label size (13px) weight 400.

## Color

Source: .impeccable.md / globals.css. CSS variable names confirmed from codebase scan.

| Role | Value | CSS Variable | Usage |
|------|-------|-------------|-------|
| Background (60%) | `#09090b` zinc-950 | `--background` | Page backgrounds, app shell, detail page backdrop |
| Surface (30%) | `#18181b` zinc-900 | `--card` | Prompt cards, filter bar background, metadata sidebar, admin form sections, popover/dropdown backgrounds |
| Accent (10%) | `#4287FF` | `--primary` / `--ring` | Reserved for specific elements listed below |
| Destructive | `#E3392A` | `--destructive` | Deprecate button, deprecation confirmation dialog, destructive action states |
| Warning | `#FFB852` | `--warning` | Deprecated status badge |
| Success | `#65CFB2` | `--success` | Active/healthy status badge |
| Text primary | `oklch(0.985 0 0)` | `--foreground` | All primary content |
| Text muted | `#71717a` zinc-500 | `--muted-foreground` | Metadata, secondary labels, placeholder text |
| Border | `#27272a` zinc-800 | `--border` | Card borders, dividers, input borders, filter bar separator |

**Accent (`#4287FF`) reserved for:**
1. "New Prompt" CTA button (admin only)
2. "Save Prompt" / "Update Prompt" primary submit buttons on admin forms
3. Active filter chip background (selected state)
4. Focus ring on all inputs and interactive controls
5. Active nav item in sidebar (Library link when on library route)
6. Copy-to-clipboard button hover state
7. Card hover border glow (`border-primary/30`)

Accent is NOT used for: category badges, model badges, star ratings, checkout counts, or decorative elements.

## Interaction Patterns

Source: CONTEXT.md decisions D-01 through D-19.

### Library Grid / List Toggle (D-01)
- Default view: card grid, 3 columns at `lg`, 4 columns at `xl` (responsive)
- Toggle button in filter bar right area — icon-only (grid icon / list icon), 36px height
- Toggle state persisted in URL param `?view=grid|list` via nuqs

### Prompt Cards (D-02, D-03)
- Card surface: `--card` (#18181b), border: `--border` (#27272a), border-radius: `--radius` (0.625rem)
- Hover state: border color transitions to `border-primary/30` — no shadow, no elevation lift
- Category badge: top-left, uses `--warning`/`--success`/muted fill by category
- Model badge: top-right, muted surface + label text
- Star rating: filled stars in `#FFB852` (warning) — semantic amber for quality signal
- Checkout count: muted text with lucide `Download` icon at 14px
- Whole card is a Next.js `<Link>` — `cursor-pointer`

### Filter Bar (D-04, D-05, D-06, D-07)
- Sticky below page header, `bg-background` with `border-b border-border`
- Left: search `<Input>` — placeholder "Search prompts..." — 300ms debounce
- Center: `<Select>` dropdowns for category, capability type, industry, target model, status
- Right: effectiveness `<Slider>` (0–5 range), sort `<Select>`, view toggle buttons
- Active filter chips row below bar: `bg-primary/10 text-primary border-primary/20`, dismiss × button on each
- "Clear all" button appears only when ≥1 filter active — ghost variant, `text-muted-foreground`
- Result count: "6 of 18 prompts" — label size (13px), muted text, right of chips row

### Detail Page Layout (D-08, D-09, D-10, D-11)
- Two-column: content column (`flex-1`, min 0) + metadata sidebar (280px fixed width)
- Gap between columns: 32px (xl)
- Back link: "← Library" at top, body size, muted text, hover primary
- Markdown content: rendered in `prose` class override using Geist Mono at 13px/1.6 line-height
- Copy button: secondary variant in sidebar, full width; icon swaps `Copy` → `Check` for 2 seconds post-click; sonner toast: "Copied to clipboard"
- Stats block: 4 items (avg rating, total checkouts, active forks, total ratings) in 2×2 grid, label text, muted labels + primary values

### Admin CRUD (D-12, D-13, D-14, D-15, D-16)
- "New Prompt" button: right-aligned in page header, primary variant, only rendered when `effectiveRole === 'admin'`
- Form pages: full-page, max-width 768px centered, lg (24px) section gaps
- Write/Preview tabs: `<Tabs>` shadcn component, Write tab = `<Textarea>` in Geist Mono; Preview tab = react-markdown render
- Save/Cancel: Save = primary button ("Save Prompt" / "Update Prompt"), Cancel = ghost button — right-aligned at bottom of form
- Deprecate: secondary-destructive button on edit page; triggers `<AlertDialog>` — see Copywriting section
- Admin controls (Edit button on detail page, Deprecate button, New Prompt button): not rendered for non-admin users — no disabled state, no hidden CSS — conditional render in JSX

### Loading States
- Library grid skeleton: 12 `<Skeleton>` cards (3×4 grid), matching card dimensions, `animate-pulse`
- Detail page skeleton: content column = 8 skeleton lines staggered width; sidebar = 4 skeleton stat blocks
- Filter bar: no skeleton — renders immediately with empty state
- Skeleton card dimensions: 100% width, 180px height

### Empty State (D-18)
- Position: centered in the grid area, vertically centered in remaining viewport height
- Icon: lucide `SearchX` at 40px, muted foreground
- See Copywriting section for copy

## Copywriting Contract

Source: CONTEXT.md decisions + .impeccable.md voice guidelines ("Direct, concise, no marketing fluff").

| Element | Copy | Notes |
|---------|------|-------|
| Primary CTA (admin create) | "New Prompt" | Button label in page header |
| Primary CTA (admin save create) | "Save Prompt" | Submit button on `/library/new` form |
| Primary CTA (admin save edit) | "Update Prompt" | Submit button on `/library/[id]/edit` form |
| Cancel action | "Cancel" | Ghost button on form pages — navigates back |
| Copy to clipboard | "Copy" | Button label in detail sidebar; icon-only on narrow sidebar |
| Copy success toast | "Copied to clipboard" | Sonner toast, 2-second duration, no action button |
| Empty state heading | "No prompts match your filters" | Centered, heading size (16px/600) |
| Empty state body | "Try adjusting your filters or clear them to see all prompts." | Body size, muted text |
| Empty state CTA | "Clear all filters" | Ghost button below empty state message |
| No library prompts (zero data) | "No prompts in the library yet." | Shown only to admins when database is empty |
| Search placeholder | "Search prompts..." | Input placeholder, muted text |
| Form error — required field | "This field is required." | Inline below field, destructive text, 13px |
| Form error — save failed | "Failed to save prompt. Please try again." | Toast (sonner), destructive variant |
| Deprecation button label | "Deprecate Prompt" | On edit page, destructive variant |
| Deprecation dialog title | "Deprecate this prompt?" | AlertDialog title |
| Deprecation dialog body | "This prompt will be hidden from the library but remains in the database. Consultants will no longer see it in browse or search results." | AlertDialog description |
| Deprecation confirm action | "Deprecate" | AlertDialog confirm button, destructive |
| Deprecation cancel action | "Cancel" | AlertDialog cancel button |
| Deprecated status badge | "Deprecated" | Warning color badge — visible on direct URL access by admin |
| Active status badge | "Active" | Success color badge — shown in metadata sidebar |
| Detail page back link | "← Library" | Navigation link text, muted, hover primary |
| Sort dropdown label | "Sort by" | Trigger label prefix before selected option |
| Sort options | "Highest rated", "Most used", "Newest first", "A–Z" | D-17 sort options |
| Result count format | "{N} of {total} prompts" | e.g. "6 of 18 prompts" — singular: "1 of 18 prompts" |
| View toggle tooltips | "Grid view", "List view" | Tooltip on icon-only toggle buttons |

**Voice rules (from .impeccable.md):**
- Labels describe what things do — not what they aspire to be
- No exclamation marks
- No "successfully" prefix — just state the outcome: "Copied to clipboard", not "Successfully copied!"
- Confirmation dialogs state consequence before action, not vice versa
